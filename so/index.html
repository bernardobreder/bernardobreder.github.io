Operating System

Assuntos em palta

Gerenciador de Classe

O gerenciador irá controlar novas classes instaladas, a sua descrição e especificação. O sistema operacional entende que cada aplicação misturado com a implementação do sistema possui um conjunto de classe versonado conhecido e gerenciado.

O sistema terá acesso a uma floresta de aplicações, incluindo o sistema, com suas descrições de classe organizada hierarquicamente na forma de pacote. Por exemplo, o sistema poderá ter instalado 3 aplicações, fazendo com que a floresta seja composta por 4 árvores, onde o primeiro corresponde ao do sistema operacional e as outras 3 das aplicações. Para as 4 cada raiz, uma hierarquia de pacote será visualizado com as classes específico para cada aplicação.

Gerenciador de Memória
O gerenciador de memória tem o papel de fazer gestão do espaço real da memória para a alocação estática para as thread criadas e dinamica para as instancia de classes a serem inicilizada. As classes devem ser alocadas pelo pool de memória para classes e objetos de tamanho dinâmico devem ser alocado por espaço de memória para objetos dinâmico.
Gerenciador de Processo
O gerenciador de processo irá controlar quais processos estão aberto em execução o que ele está utilizando.
Gerenciador de Thread
Gerenciamento de Thread criado pelos processos com sua memória estático previamente alocada.
Banco de Dados

Sistema de banco de dados estruturado compartilhado para os processos para gerenciar e administrar acesso aos dados de aplicação instaladas.

Gerenciador de Transação
A gerencia de transação tem o papel de gerenciar marcos de mudanças atomicas nos dados do sistema e aplicação de tal forma a garantir o isolamento de transação, restauração caso haja erro e recuperação dos dados em falha total do sistema.
Gerenciador de Notificação
Gerencia de notificação entre sistemas, apicações e hardware.
Gerenciador de Atualização
Gerencia a atualização do sistema, aplicações, banco de dados e cache
Linguagem de Programação

Linguagem de programação apropriada para os requisitos do sistema adaptado as ferramentas.

Lambda
A linguagem de programação deve ter recurso de lambda para conversão e manipulação de dados para auxiliar o desenvolvimento. Esse recurso deve fazer bom uso de memória e processamento paralelo se for preciso.
Ambiente de Desenvolvimento
Criar um ambiente de desenvolvimento para auxiliar o desenvolvimento de apicações para essa plataforma
Coletor de Lixo
Coletor de objetos não mais usado e liberá-lo para a gerencia de memória. Esse coletor pode ser implementado por contador de referencia, mark-and-scan para ilhas de objetos e cache antigos.
Gerenciador de Lock
O gerenciador de lock tem o papel de dispor de ferramentas de lock para o sistema e aplicações e também a gerencia de deadlock.
Gerenciador de Modificação
A gerencia de moficação tem a responsabilidade de gerenciar as mudanças dos dados do sistema e aplicação de tal forma a minimizar o espaço de armazenamento e de gestão de mudanças. Essa gerencia reduz muito o espaço de armazenamento quando é apregado adequadamente e possibilita uma gestão das modificações efetuadas possibilitando a reversão, merge e entre outras operações em cima de arquvios.
Prolog
A máquina de prolog pode ser uma ferramenta importante para melhor as aplicações do sistema com base no que faz melhor.
Mineração de Dados
A mineração de dados de todas as aplicações pode auxiliar na aplicação tornando as aplicações mais proativa.
Rede Neurais
A Rede Neurais aplicada nas aplicações pode auxiliar em aplicações mais proativa.
Segurança
Criptografia de dados, Resumo de Arquivo, Transferencia de Dados, Big Integer, Protocolo SSL, Assinatura digital
Sistema de Arquivo Estruturado
O sistema de arquivo de forma textual e estruturado permite ter um melhor espaço de armazenamento em função da gerencia de modificação e da abstração com que é feito o conteúdo na aplicação. Os arquivos devem armazenar somente o que o usuário precisa para preencher o documento de uma aplicação.
Gerencia de Mensagem
Uma gerenciar de mensagem para troca de mensagem entre usuários da tal forma a facilitar a comunicação. Grupos de assuntos devem ser levado em consideração para melhorar a organização dessas mensagem. Um conjunto de features para gerencia de mensagem pelo administrador do grupo pode ser apropriado para que as mensagem fiquem bem organizadas e de facil consulta.
Serialização de Objetos
Serialização de qualquer objetos para permitir que os processos descanse quando necessário. Para objetos que não podem ser serializado, uma inicialização e entrada no estado de descanso deve ser implementado para possibilitar a liberação da aplicação da memória.
Gerenciador de Tarefas
Uma gerencia de tarefas deve ser bem contruida pensando em um compartilhamento com todos os processos, onde as tarefas possui a descrição de sua prioridade e de onde irá impactar a sua atualização ou execução. Dependendo do tipo de tarefa como de interface, a atualização pode ser postergada em função da aplicação estar minimizada.
App Package

Gestão de Aplicações instaladas no sistema e controle de acesso ao sistema. Esse tópico traz a responsabilidade para a aplicação só usar os dados do usuário se ele deu permissão para isso.

Sandbox
Sistema de arquivo interno com total acesso aos dados e um acesso restrito ao sistema de arquivo externo a aplicação controlando tudo que é lido e modificado, podendo verificar e reverter todas as mudanças efetuada pela aplicação.
Compress
Compressão de dados com o objetivo de reduzir o espaço de armazenamento em função da coleção de dados existente no sistema de arquivo. Analisar o que poderá ser feito para comprimir todos os dados de texto considerando a mudança da tabela de index com o tempo.
Imprimir
A exportação para papel é um recurso importante para retirar a informação do meio digital para o meio real.
Sound
Gestão de musica e som do sistema.
Calendario
Gestão de data, calendario, mês, formatação, timezone, férias.
Collection
Estrutura de dados de coleção de dados otimizado com a gerencia de memória.
Algoritmos
Algoritmos de biblioteca uteis para o sistema e aplicações melhorando o reuso de código.
Concorrencia
Biblioteca de classes de controle de concorrencia para reuso pelo sistema e pela aplicação.
Log
Gerenciamento de log para depuração e entendimento das mudanças ocorridas. É importante que a ferramenta seja util para investigação de falhas ocorridas.
Test
Arquitetura de Teste para as aplicações com o objetivo de melhor a qualidade das aplicações nela desenvolvida.
Validation
Biblioteca de validação de dados e parametros com o objetivo de melhor a qualidade das aplicações nela desenvolvida.
Regex
Biblioteca de Regex com o objetivo de melhor a forma como é feita consulta nas aplicações.

Componentes gráficos

User Preference
Preferencia de usuário para que as aplicações possam ser fechadas e reabertas com o mesmo estado anterior, fazendo com que seja possível liberar a memória de uma aplicação e restauração sem afetar a interface do usuário.
Codec
Biblioteca de Codec como Base64, Audio, Zip, Cripto, ...
WebService
Biblioteca de webservice para auxiliar aplicaçoes que usa dados da internet para fonte de dados.
Structure Service
Biblioteca de serviço proprietário da plataforma de tal forma a ficar mais facil de desenvolver com essa plataforma.
Internacionalization
Internacionalização das aplicações com o objetivo de ficar mais próximo a realidade do usuário.
Cache
Infraestrutura de cache auxiliado pela gerencia de memória
